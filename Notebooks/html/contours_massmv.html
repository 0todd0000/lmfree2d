Mass-multivariate analysis of 2D contour data in Python¶

This notebook...

  • ... provides an overview of mass-multivariate, two-sample hypothesis testing of 2D contour data in Python
  • ... explains the details of script contours_massmv_single.py which appears in this repository
  • ... is limited to mass-multivariate analysis (i.e., multivariate test statistics calculated at each point, with inference conducted in an omnibus sense over all points)
  • ... is directed at novice Python users, who may be using Python for the first time.
  • ... is likely not useful for intermediate or advanced Python users; please refer isntead to the scripts in ./lmfree2d/Python/

Shuffle data¶

To represent general datasets, and also to remove and/or test for bias associated with a particular point ordering, let's shuffle the data points around so that they no longer appear in order, as in the figure above. This can be achieved with the shuffle command:

plt.figure(figsize=(10,4))
ax = plt.axes()

Note that the points now appear in a random order.

Conduct point set registration¶

Next let's register (i.e., spatially align) the contours. While landmark techniques typically achieve spatial registration using GPA, this requires both (a) an identical number of points in all shapes, and (b) that all points are ordered in a homologous manner. As we see in the figures above, the shuffled Bell dataset satisfies neither (a) nor (b).

Let's instead use the coherent point drift (CPD) algorithm to register the shapes. Note that CPD can be regarded as a generalization of GPA to cases (a) and (b). In lmfree2d, CPD can be conducted using the register_cpd function as follows:

In [7]:
r_template,n,i = lm.get_shape_with_most_points(r) # choose a template shape
r1             = lm.register_cpd(r, r_template)   # register all shapes using the CPD algorithm

print(n, i)
print( r[i].shape )


fig,AX  = plt.subplots(1, 2, figsize=(10,4))
lm.plot_registration(AX[0], r, r_template)
lm.plot_registration(AX[1], r1, r_template)
AX[0].set_title('Original')
AX[1].set_title('Registered')
plt.show()

Notes:

  • The template shape is the shape to which the other shapes (the sources) are aligned.
    • The template is fixed (stationary).
    • Sources are transformed (position, rotation and scale) to optimally match the template.
  • In this case the shape with the most number of points is selected as the template.
  • In general, different templates should be tested to ensure that final results are not affected by the choice of template.
  • The variables n and i represent the number of template points, and the index of the template shape, respectively.
  • Following CPD registration, the shapes are clearly better aligned.

Achieve correspondence¶

Correspondence algorithms aim to match homologous locations across shapes. If all contour points represent homologous locations across all shapes, the shapes are said to be in correspondence.

There are many different algorithmic approaches for achieving correspondence. Here a very simple optimum roll approach is used. This approach simply rolls the starting point around a source contour, until the source shape optimally matches a template shape. Here 'optimal' implies that the sum of squared distances between the source and template points is minimal.

In [9]:
r_template_new = r2[i]                                # get the new template (with ordered points)
r3             = lm.set_npoints(r2, n)                # set a constant number of contour points across all shapes
r4             = lm.corresp_roll(r3, r_template_new)  # run the optimum roll correspondence algorithm



fig,AX  = plt.subplots(2, 1, figsize=(10,8))
lm.plot_correspondence(AX[0], r3, r_template_new)
lm.plot_correspondence(AX[1], r4, r_template_new)
AX[0].set_title('Ordered')
AX[1].set_title('Corresponding')
plt.show()

In the figure above, purple lines are correspondence lines, connecting points in the source and template contours. Note that the correspondence algorithm, while simple, has dramatically improved correspondence.

Conduct hypothesis test¶

Now that the contour shapes are (i) spatial registered, and (ii) in correspondence, it is possible to conduct mass-multivariate hypothesis testing. Here we use spm1d for this purpose. The goal will be to compare the mean shapes from two groups, where the first group consists of the first five shapes and the second group consists of the last five shapes. Briefly, spm1d proceeds as follows:

  • Calculates the mean location of each contour point for both groups
  • Calculates the variance (acutually, covariance) surrounding each mean contour point
  • Calculates the test statistic value (the Hotelling's $T^2$ value) at each contour point, which effectively represents the ratio between (i) the mean location difference, and (ii) the location variance.
  • Conducts statistical inference (i.e., calculates a p value) across all contour points. This process effectively involves (i) calculating the distribution of $T^2$ values that would be expected by random sampling, given both truly zero difference in mean shapes, and the observed covariance, then (ii) comparing the observed $T^2$ distribution to the one expected by chance.
  • More details about these calculations, and why they are statistically valid, are provide in the spm1d and rft1d documentation.
  • Why "1D" in spm1d? Because a contour can be regarded as a 1D continuum spanning locations $u$=0 and $u$=1, where $u$ represents contour location. At each contour location there is a 2D position vector. The location $u$=0.5 represents the location half-way around the contour from the stating location ($u$=0), and the locations $u$=0 and $u$=1 represent the same location.

All of this can be achieved in lmfree2d as follows:

In [10]:
rA      = r4[:5]  # assign the first 5 shapes to one group
rB      = r4[5:]  # assign the last 5 shapes to a second group
results = lm.two_sample_test(rA, rB, parametric=True, alpha=0.05) # conduct a two-sample hypothesis test


plt.figure(figsize=(10,4))
ax = plt.axes()
results.plot(ax)
ax.legend()
plt.show()

Notes:

  • The mean shapes for each group are plotted
  • Dots represent contour points that exceed the critical threshold at $\alpha=0.05$
  • The p value represents the probability that random sampling would produce a maximum shape difference as large as the observed shape difference, if the two groups actually has identical means, given the observed shape variance.
  • If there are suprathreshold points present, then the p value is at most $\alpha$
